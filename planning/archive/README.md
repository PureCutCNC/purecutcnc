# planning/archive/

Historical plans, bug analyses, and superseded designs. **Agents: do not read files here unless you have a specific reason** — they are kept for git history and occasional human reference, not for ongoing work.

Most files here fall into one of:

- **Shipped feature plans** — the described feature is implemented; the plan is documentation of the path taken.
- **Bug analyses / debug notes** — root-cause writeups for issues that have since been fixed (or moved on).
- **Superseded plans** — replaced by a newer doc still in [`../`](../) (e.g. `TABLET_UX_Implementation_Plan.md` superseded by `TABLET_UX_COMBINED_PLAN.md`).

If you find yourself needing one of these regularly, that's a signal to promote it back into [`../`](../) and add it to [`../INDEX.md`](../INDEX.md).
